# Стиль 1-страничников ▸ Гайд v2

TL;DR ▸ принципы обязательны → структура гибкая → применяй что подходит.

---

## Философия

Это **набор инструментов**, не шаблон. Не изобретай разделы — сжимай то, что есть.

**Обязательно:**
- Экстремальная сжатость
- Символы вместо слов
- Русская основа
- Таблицы/списки вместо текста

**Опционально:**
- Конкретные разделы (KPIs, риски, шаги)
- Применяй только если есть в оригинале или органично вытекает

---

## ОБЯЗАТЕЛЬНЫЕ принципы

### 1. Экстремальная сжатость
Не предложения — фрагменты. Не абзацы — списки/таблицы. Каждое слово на вес золота.

### 2. Информационная плотность
Факты, цифры, действия. Ноль воды, преамбул, вводных конструкций.

### 3. Визуальная структура
Разметка = грамматика. Символы заменяют связки.

### 4. Русская основа + устоявшиеся термины
• Базис — русский язык
• Только общепринятые термины: KPI, CEO, CRM, API
• Сокращения по необходимости: З1/З2, ДЗ, гр, мес, н/д

---

## Обязательное форматирование

| Элемент | Формат | Пример |
|---------|--------|--------|
| **Заголовок** | `# Тема ▸ подзаголовок` | `# Рост-сообщества ▸ процесс` |
| **TL;DR** | Одна строка, символы | `TL;DR ▸ 2 звонка → выбор роли → запуск` |
| **Время** | Сокращения | `60-90′` `+1-3д` `2-4ч/н` `1-2 мес` |
| **Метрики** | Точка · разделитель | `З1→З2 70% · З2→роль 80%` |
| **Списки** | `•` или `1)` | Не глубже 1 уровня |
| **Ссылки** | `[текст](путь)` или `` `файл.md` `` | Существующие — кликабельно, планируемые — кавычки |

---

## Символы-операторы (обязательно)

```
→  затем, приводит к (горизонтальный flow)
▸  раскрытие, детализация
⇒  превращение, трансформация (сильная связь)
⇢  митигация, противодействие
·  разделитель в строке (вместо ", ")
```

**Примеры:**
- `Звонок-1 → ДЗ → Звонок-2 → Запуск`
- `TL;DR ▸ суть процесса`
- `Участник ⇒ 4-мес цикл`
- `Низкий вау ⇢ артефакт сначала`
- `KPI1 70% · KPI2 80%`

---

## Язык (обязательно)

✅ **Используем обдуманно**
- Только общепринятые термины: KPI, CEO, CRM, API, IT, PR
- Сокращения: З1/З2, ДЗ, МО, гр, мес, н, ч/д
- Русские термины приоритетны: "выбор роли", "инструменты", "быстрый запуск"

✅ **Русская основа**
- Связки: "затем", "или", "для"
- Глаголы: "делаем", "запускаем", "проверяем"

❌ **Избегаем**
- Излишние англицизмы вместо русских аналогов
- Длинные предложения > 12 слов
- Пассивный залог: "было сделано", "является"
- Вводные конструкции: "таким образом", "в связи с этим"
- Абзацы > 3 строк

## ОПЦИОНАЛЬНЫЕ элементы

Применяй только если есть в исходном документе или логично вытекает из контента.

### Навигация
```markdown
**Navigate:**
- [Ссылка 1](./путь.md)
- [Ссылка 2](./путь.md)
```
Используй если: документ часть системы файлов, есть связанные документы.

### TL;DR
```markdown
TL;DR ▸ ключ → ключ → результат.
```
Используй если: можно сжать суть в одну строку (рекомендуется почти всегда).

### Таблицы процессов
```markdown
| Шаг | Когда | Цель | Детали |
|-----|-------|------|---------|
```
Используй если: есть пошаговый процесс, временная последовательность.

### KPIs / Метрики
```markdown
## KPIs
Метрика1 X% · Метрика2 Y% · Метрика3 Z%.
```
Используй если: в документе есть числовые показатели, целевые значения.

### Риски⇢Смягчение
```markdown
## Риски⇢Смягчение
Проблема ⇒ решение.
```
Используй если: в оригинале есть риски, препятствия, проблемы.

### Следующие шаги
```markdown
## Следующие шаги
1) Действие. 2) Действие. 3) Действие.
```
Используй если: в документе есть action items, план действий, этапы реализации.

### Сценарии / Варианты
```markdown
• Вариант А ⇒ шаг1 → шаг2 (ориентир).
• Вариант Б ⇒ ...
```
Используй если: несколько путей, развилки, альтернативы.

## Ссылки на файлы (обязательно)

**Правило:**
- **Существующий файл** → `[текст](./путь/файл.md)` (кликабельно)
- **Планируемый файл** → `` `файл.md` `` (в обратных кавычках)
- **НЕ** создавай ссылки на несуществующие файлы

**Примеры:**
```markdown
# ✅ Правильно
См. [инструкцию call_1](./playbooks/call_1.md)  ← файл существует
[ ] Создать: `call_2.md`  ← файл не создан

# ❌ Неправильно
См. инструкцию call_1.md  ← не кликабельно
[Инструкция](./call_2.md)  ← файла не существует!
```

## Процесс работы

1. **Читай оригинал** → какие разделы есть?
2. **Сжимай контент** → применяй принципы (символы, таблицы, фрагменты)
3. **Не изобретай** → нет рисков в оригинале? не добавляй "Риски"
4. **Проверяй** → каждый раздел оправдан? есть в источнике?

## Anti-примеры vs Правильно

❌ **Плохо (добавил лишнее):**
```
# Заметки со встречи ▸ 1-страница

TL;DR ▸ обсудили проект X.

## Основное
- Решили запустить тест
- Срок: 2 недели

## Риски⇢Смягчение  ← ЛИШНЕЕ! в заметке их не было
Задержки ⇒ буфер времени

## Следующие шаги  ← ЛИШНЕЕ! в заметке их не было
1) Подготовить план. 2) Созвониться.
```

✅ **Правильно (только то, что есть):**
```
# Заметки со встречи ▸ проект X

TL;DR ▸ запуск теста → 2 недели.

• Решили запустить тест функции Y
• Срок: 2 н
• Ответственный: Иван
```

---

❌ **Плохо (длинно, вода):**
```
В первую очередь нам необходимо провести первый звонок,
который обычно длится от 60 до 90 минут. На этом звонке
мы будем создавать первый артефакт.
```

✅ **Правильно (сжато, символы):**
```
Звонок-1 | 60-90′ | Искра
• 1 артефакт в реальном времени → ценность
• 2 пути: Участник vs Ведущий
```
